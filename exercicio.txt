EXERCÍCIO 1 — Estrutura do Projeto SGTA
========================================

models.py  → Define os modelos (tabelas do banco de dados); no SGTA contém o modelo Tarefa com campos como titulo, status, prioridade e data_entrega.

views.py   → Contém as funções que processam as requisições HTTP e retornam respostas JSON para o cliente.

urls.py    → Mapeia os endereços (URLs) da API para as views que devem ser executadas em cada rota.

admin.py   → Registra os modelos no painel administrativo do Django, permitindo gerenciar os dados via interface web.

apps.py    → Configura e identifica o app dentro do projeto Django, sendo referenciado em INSTALLED_APPS no settings.py.
